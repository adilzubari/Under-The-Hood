# Support & Sponsor

**Under The Hood is free, open, and built as a gift to the community.** No paywall, no email capture, no ads. If it helps you prepare for interviews, refresh a concept at work, or onboard a teammate — that's the whole goal.

This file exists for two audiences:

1. **You need help with the content** — see [Getting help](#-getting-help).
2. **You want to support the project** — see [Supporting the project](#-supporting-the-project).

---

## 🆘 Getting help

**Free, always:**

- 🐛 **Found a mistake or broken link?** → [Open a correction issue](https://github.com/adilzubari/Under-The-Hood/issues/new?template=correction.md)
- 💡 **Want a topic covered?** → [Open a topic request](https://github.com/adilzubari/Under-The-Hood/issues/new?template=new-topic-request.md)
- 🏗️ **Suggest a scenario?** → [Open a scenario request](https://github.com/adilzubari/Under-The-Hood/issues/new?template=scenario-request.md)
- 💬 **General questions or discussion?** → [GitHub Discussions](https://github.com/adilzubari/Under-The-Hood/discussions) *(enable if not already live)*

**Please don't:** open issues for content requests that are already listed as "suggested next" in the phase indexes — those are already on the roadmap.

## 🧡 Supporting the project

This project will always be **free to read, copy, fork, and share** under the MIT license. Support is entirely optional and has zero effect on content access.

### Ways to help that cost you nothing

These are the **most valuable** ways to help:

- ⭐ **Star the repo.** GitHub ranks repos by stars in search. One click = helps a stranger find this.
- 🔁 **Share it.** Post on LinkedIn / X / Reddit / your team Slack. Most users arrive through a personal recommendation.
- ✍️ **Contribute.** Fix a typo, add a topic, write a scenario — see [CONTRIBUTING.md](CONTRIBUTING.md). Every PR is welcome.
- 🎯 **Use it.** Report what worked, what didn't. Real usage drives better content.

### What happens to support funds?

There's no team and no overhead. Any funds go toward:

1. **Time to keep writing** — filling Phase 2-20, adding scenarios, maintaining quality.
2. **Reference materials** — books and courses that deepen content (every citation in the References sections was read by me; good books aren't cheap).
3. **Tooling / services** — domain, any future hosting (not currently needed; repo is pure GitHub).

Nothing is promised; no rewards, no perks, no exclusive content. If you tip, you're paying for content you've already received. If you don't, the content still works exactly the same.

## 🌍 Why is this free?

> Most of us got into software through free resources: docs, blog posts, Stack Overflow, open-source code. When you can give back, you should.

The knowledge in this repo comes from a decade+ of reading, writing, shipping, and interviewing. Locking it behind a paywall would put it out of reach for exactly the people who need it most — students, early-career engineers, people switching fields, engineers in countries where $200 courses aren't realistic.

**Free by default. Donate if it helped. Share if you can.**
