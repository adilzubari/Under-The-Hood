# Phase 20 — Soft Skills & System Design

> **Phase abstract:** The non-code skills that distinguish senior engineers. Communicating trade-offs, mentoring, debugging culture, postmortems, and the architecture-decision-making muscle.

📝 **Status: stub.**

## Topics

- [ ] **Communicating design trade-offs** — naming alternatives, articulating costs, decision records (ADRs).
- [ ] **Estimating scalability & cost** — back-of-envelope numbers. Latency rule-of-thumb.
- [ ] **Reviewing & mentoring code** — what to focus on. The art of the kind, useful comment.
- [ ] **Designing for change & modularity** — boundaries that absorb future change. YAGNI vs over-design.
- [ ] **Documentation & readability** — code comments vs READMEs vs ADRs vs runbooks.
- [ ] **Debugging production issues** — bisecting, reproducing, hypothesis-driven debugging.
- [ ] **Root cause analysis (RCA)** — five whys. Avoiding "human error" as the conclusion.
- [ ] **Postmortems & learning culture** — blameless postmortem template. Action items that actually happen.
- [ ] **Stakeholder communication** — translating tech to non-tech. Status updates that respect time.
- [ ] **Prioritization & architecture decision-making** — when to invest in foundations vs ship.

## Suggested learning order

1. ADRs (the underrated skill)
2. Back-of-envelope estimation (interview gold; saves arguments)
3. Postmortem template + RCA discipline (cultural foundation)
4. Code review patterns (asynchronous mentoring at scale)
